# PiClaw 使用教程

## 适用人群

PiClaw 适合希望把 AI 编程助手部署在懒猫微服上的开发者。它提供浏览器聊天入口、会话管理、搜索和持久化工作区，适合长期项目、自动化脚本、代码阅读和日常开发辅助。

## 安装后第一步

1. 在懒猫微服中安装 PiClaw。
2. 打开 `https://piclaw.<你的盒子域名>/`。
3. 确认页面进入 PiClaw 主界面，而不是“启动中”、空白页、平台错误页或 5xx 错误页。
4. 首次使用时打开设置，配置需要使用的 AI provider/model。

## 核心流程

1. 在主界面的消息输入框输入一个开发任务，例如让代理阅读项目 README、解释某个脚本，或生成一个小型修改计划。
2. 通过会话按钮确认当前代理是 `@piclaw`，必要时新建或切换会话。
3. 发送消息后，观察回复是否进入当前会话；需要排查历史任务时使用搜索入口。
4. 把长期项目文件放在持久化工作区中，后续重新打开应用时继续使用同一个上下文。

## 模型配置提示

- PiClaw 默认进入主界面后可能提示实例还未完成设置，这是正常的首次配置状态。
- 用户需要在设置页填入自己的模型供应商、模型名或相关凭据。
- 不要在公开截图里暴露 API key、token、账号信息或私有项目路径。

## 常见问题

- 如果页面停在“启动中”，优先检查 `pibox` 容器日志和 supervisor 日志。
- 如果主界面能打开但无法发送消息，先检查是否已经完成模型配置。
- 如果页面能打开但工作区不可写，检查 `/workspace` 与 `/config` 的挂载和 owner。
- 如果功能需要外部模型 API，请在设置页配置，不要把凭据写进教程或公开文案。
